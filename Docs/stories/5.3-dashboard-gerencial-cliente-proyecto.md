# Story 5.3: Dashboard Gerencial por Cliente, Proyecto y Carga

Status: draft

## Story

As a gerente,
I want visualizar horas por cliente, proyecto y consultor,
So that pueda gestionar capacidad y priorizar decisiones operativas.

## Acceptance Criteria

1. **Given** un Gerente autenticado, **when** abre el dashboard gerencial, **then** ve distribución de horas por cliente y proyecto **and** ve indicadores de carga de trabajo por consultor.

2. **Given** filtros de período aplicados, **when** el Gerente cambia el rango, **then** las métricas y gráficos se recalculan correctamente **and** las consultas respetan índices para rendimiento.

3. **Given** un usuario sin rol Gerente o Admin, **when** intenta acceder al dashboard gerencial, **then** el API rechaza el acceso **and** la UI no muestra la sección no permitida.

## Tasks / Subtasks

### Backend

- [ ] Crear `Application/Features/Dashboard/Queries/GetDistribucionPorCliente/` — horas agrupadas por Cliente y Proyecto con Dapper
- [ ] Agregar endpoint `GET /api/dashboard/gerencial?desde={}&hasta={}` (Gerente + Admin)

### Frontend

- [ ] Agregar sección `/dashboard` visible solo para Gerente/Admin con:
  - Dos gráficos ApexCharts: horas por Cliente (pie/donut) y horas por Proyecto (bar)
  - KpgStatCard: total horas período, total clientes activos, top cliente

### Verificación

- [ ] Build limpio
