## Dev Notes

### Stack implementado

- Backend: .NET 10, C# 13, ASP.NET Core, SQL Server 2019+
- ORM: EF Core 10 (escritura) + Dapper 2.x (lecturas/reportes)
- CQRS: MediatR con vertical slices bajo `Application/Features/`
- Frontend: Blazor WebAssembly (.NET 10), MudBlazor
- Scaffolding híbrido: Jason Taylor CleanArchitecture para backend + `dotnet new blazorwasm`

### Estructura de proyectos

```
Backend/
  src/
    Domain/          → Entidades, excepciones de dominio, constantes
    Application/     → CQRS (Features), interfaces, validadores
    Infrastructure/  → EF Core, Dapper, Identity, repositorios
    Api/             → Endpoints mínimos, IEndpointGroup pattern
  tests/
    Domain.UnitTests/
    Application.UnitTests/
    Infrastructure.IntegrationTests/

Fronted/
  src/
    WebUI/           → Blazor WASM (proyecto oficial V1)
  (raíz)            → Proyecto Angular legacy (prototipo, ignorar)
```

### Central Package Management

`Directory.Packages.props` controla versiones. Los `.csproj` referencian sin `Version=""`.

### IEndpointGroup pattern

Cada módulo tiene una clase estática con `RoutePrefix` y método `Map(RouteGroupBuilder)`. Auto-registrado vía reflexión en `Program.cs`.
