# Story 1.3: Sesión Segura con Refresh Token y Logout

Status: done

## Story

As a usuario autenticado,
I want que mi sesión se renueve de forma segura y pueda cerrarla explícitamente,
So that no pierda trabajo durante el uso normal y pueda terminar mi sesión cuando corresponda.

## Acceptance Criteria

1. **Given** un usuario autenticado con refresh token vigente, **when** el JWT está por expirar o la app recarga, **then** el sistema solicita renovación al endpoint de refresh, **and** recibe un nuevo JWT si el refresh token no está expirado ni revocado.

2. **Given** un usuario autenticado, **when** cierra sesión, **then** el refresh token activo queda revocado en servidor, **and** el JWT se elimina de memoria del cliente.

3. **Given** la política de seguridad aprobada, **when** el cliente almacena tokens, **then** el JWT vive exclusivamente en memoria, **and** no se guarda en localStorage ni sessionStorage.

## Dev Notes

### Flujo de refresh

1. `KpgAuthStateProvider` detecta que el JWT expira en < 5 minutos
2. Llama `POST /api/auth/refresh` con el refresh token actual
3. Servidor verifica que no está expirado ni revocado
4. Revoca el refresh token actual (single-use)
5. Genera nuevo JWT (60 min) + nuevo refresh token (8h)
6. Cliente actualiza JWT en memoria
7. Si falla → logout automático

### Seguridad de tokens

- JWT: solo en memoria del `KpgAuthStateProvider` (campo `_token`)
- Refresh token: guardado en `HttpOnly` cookie o en memoria según implementación
- NFR7: nunca en localStorage/sessionStorage
- NFR8: JWT max 60 min, refresh max 8h

### Logout

- Frontend: limpia `_token` en `KpgAuthStateProvider`, notifica cambio de estado
- Backend: marca `RefreshToken.IsRevoked = true`, `RevokedAt = DateTime.UtcNow`
