# Story 5.2: Distribución de Horas del Equipo para Supervisor

Status: draft

## Story

As a supervisor,
I want ver la distribución de horas de mi equipo por consultor,
So that pueda entender carga y avance del período.

## Acceptance Criteria

1. **Given** un Supervisor autenticado, **when** selecciona un período (día, semana, mes o rango personalizado), **then** ve distribución de horas por consultor de su equipo **and** no ve datos de usuarios fuera de su alcance.

2. **Given** datos disponibles para el período, **when** el dashboard renderiza visualizaciones, **then** usa `KpgStatCard` (horas totales) y gráfico ApexCharts de barras por consultor.

3. **Given** no hay registros en el período, **when** el Supervisor consulta la distribución, **then** se muestra estado vacío específico **and** no se renderizan gráficos engañosos.

## Tasks / Subtasks

### Backend

- [ ] Crear `Application/Features/Dashboard/Queries/GetDistribucionHoras/GetDistribucionHorasQuery.cs`
  - Parámetros: `DateOnly Desde`, `DateOnly Hasta`
  - Autorización: Supervisor, Gerente, Admin
- [ ] Crear handler con Dapper:
  ```sql
  SELECT ISNULL(u.NombreCompleto, u.Email) AS Nombre,
         SUM(DATEDIFF(MINUTE, r.HoraEntrada, r.HoraSalida)) / 60.0 AS TotalHoras
  FROM   RegistrosHoras r
  JOIN   AspNetUsers u ON r.UserId = u.Id
  WHERE  r.FechaRegistro BETWEEN @Desde AND @Hasta
  GROUP  BY u.Id, u.NombreCompleto, u.Email
  ORDER  BY TotalHoras DESC
  ```
- [ ] Agregar endpoint `GET /api/dashboard/distribucion-horas?desde={}&hasta={}` a `Dashboard.cs`

### Frontend

- [ ] Agregar sección "Distribución del período" a `DashboardPage.razor`
  - Selector de período: MudButtonGroup (Hoy / Semana / Mes / Personalizado)
  - Para "Personalizado": dos MudDatePicker (Desde / Hasta)
- [ ] Agregar `GetDistribucionHorasAsync(DateOnly desde, DateOnly hasta)` a `IDashboardRepository`
- [ ] Integrar `ApexChart` (BarChart horizontal) con datos por consultor
  - Usar `Blazor-ApexCharts` package ya instalado
  - Eje Y: nombres de consultores; Eje X: total de horas

### Verificación

- [ ] Build limpio backend y frontend
- [ ] Verificar manualmente: gráfico muestra horas del período seleccionado

## Dev Notes

### ApexCharts setup

Blazor-ApexCharts ya está instalado. Patrón básico:
```razor
<ApexChart TItem="DistribucionItem" Title="Horas por Consultor">
    <ApexPointSeries TItem="DistribucionItem"
                     Items="_distribucion"
                     Name="Horas"
                     XValue="@(e => e.Nombre)"
                     YValue="@(e => (decimal)e.TotalHoras)"
                     SeriesType="SeriesType.Bar" />
</ApexChart>
```

### Cálculo de horas

Las horas se calculan en la query SQL como `DATEDIFF(MINUTE, HoraEntrada, HoraSalida) / 60.0`. Retornar como `decimal` con 1 decimal de precisión.

### Período por defecto

Al cargar la página por primera vez, mostrar "Esta semana" (lunes al domingo de la semana actual).
