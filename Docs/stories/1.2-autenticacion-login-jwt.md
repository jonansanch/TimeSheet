# Story 1.2: Autenticación con Login y JWT

Status: done

## Story

As a usuario interno,
I want iniciar sesión con mis credenciales,
So that pueda acceder de forma segura a KPG Timesheet.

## Acceptance Criteria

1. **Given** un usuario activo con credenciales válidas, **when** envía usuario y contraseña desde la pantalla de login, **then** el API valida credenciales con ASP.NET Core Identity, **and** retorna un JWT firmado y un refresh token válido.

2. **Given** credenciales inválidas o usuario inactivo, **when** el usuario intenta iniciar sesión, **then** el sistema rechaza el acceso con Problem Details, **and** no expone información sensible sobre la causa exacta.

3. **Given** la pantalla de login corporativo, **when** se muestra al usuario, **then** incluye identidad KPG, usuario, contraseña, botón Iniciar sesión y texto de recuperación orientado a contactar al Admin.

## Dev Notes

### Seguridad JWT — restricción crítica

- JWT se almacena **exclusivamente en memoria** del `KpgAuthStateProvider`
- **Nunca** en `localStorage` ni `sessionStorage` (NFR7)
- JWT expira en máximo 60 minutos (NFR8)
- Contraseñas con hash bcrypt vía Identity (NFR10)
- HTTPS obligatorio (NFR6)

### ApplicationUser

```csharp
public class ApplicationUser : IdentityUser
{
    public string NombreCompleto { get; set; } = string.Empty;
    public bool Activo { get; set; } = true;
    public ICollection<RefreshToken> RefreshTokens { get; set; } = [];
}
```

### Tabla RefreshTokens

Columnas: Id, UserId (FK), Token, Expires, IsRevoked, Created, RevokedAt.
