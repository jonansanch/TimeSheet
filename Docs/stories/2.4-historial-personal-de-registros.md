# Story 2.4: Historial Personal de Registros

Status: done

## Story

As a empleado,
I want consultar mi historial de registros,
So that pueda revisar qué horas he reportado.

## Acceptance Criteria

1. Dado un empleado autenticado, cuando abre "Mis Registros", entonces ve únicamente sus propios registros, y los registros muestran fecha, turno, horas, cliente, proyecto, modalidad y descripción.
2. Dada una lista con múltiples registros, cuando el empleado ordena o filtra por fecha, entonces la tabla actualiza los resultados correctamente y mantiene formato `dd/MM/yyyy` para fechas y `HH:mm` para horas.
3. Dado el empleado no tiene registros, cuando abre "Mis Registros", entonces ve un estado vacío específico y puede iniciar registro de horas desde esa vista.

### Query Handler — Patrón de Implementación

Seguir exactamente el patrón de `GetRegistrosRecientesQueryHandler.cs`:

```csharp
public class GetMisRegistrosQueryHandler : IRequestHandler<GetMisRegistrosQuery, IEnumerable<MisRegistrosItemDto>>
{
    private readonly IApplicationDbContext _context;
    private readonly IUser _user;

    public GetMisRegistrosQueryHandler(IApplicationDbContext context, IUser user)
    {
        _context = context;
        _user = user;
    }

    public async Task<IEnumerable<MisRegistrosItemDto>> Handle(
        GetMisRegistrosQuery request,
        CancellationToken cancellationToken)
    {
        var userId = _user.Id;
        if (string.IsNullOrWhiteSpace(userId))
            return [];

        return await _context.RegistrosHoras
            .Where(r => r.UserId == userId)
            .OrderByDescending(r => r.FechaRegistro)
            .ThenBy(r => r.Turno)
            .Select(r => new MisRegistrosItemDto(
                r.Id,
                r.FechaRegistro,
                r.Turno,
                r.HoraEntrada,
                r.HoraSalida,
                r.Cliente,
                r.Proyecto,
                r.Modalidad,
                r.Descripcion))
            .ToListAsync(cancellationToken);
    }
}
```

```csharp
namespace KPG.Timesheet.Infrastructure.IntegrationTests.RegistroHoras;

public class GetMisRegistrosQueryHandlerTests
{
    [Fact]
    public async Task Handle_WhenUserHasNoRegistros_ShouldReturnEmpty()
    {

        var handler = new GetMisRegistrosQueryHandler(context, new TestUser("user-sin-registros"));

        var result = await handler.Handle(new GetMisRegistrosQuery(), CancellationToken.None);

        result.Should().BeEmpty();

    [Fact]
    public async Task Handle_WhenUserHasRegistros_ShouldReturnOnlyTheirOwnRecords()
    {
        await using var context = CreateContext();
        context.RegistrosHoras.Add(MakeRegistro("user-1", "KPG", "Timesheet", new DateOnly(2026, 5, 10)));
        context.RegistrosHoras.Add(MakeRegistro("user-2", "Otro", "Otro Proyecto", new DateOnly(2026, 5, 10)));
        await context.SaveChangesAsync(CancellationToken.None);

        var handler = new GetMisRegistrosQueryHandler(context, new TestUser("user-1"));
        var result = (await handler.Handle(new GetMisRegistrosQuery(), CancellationToken.None)).ToList();

        result.Should().HaveCount(1);
        result[0].Cliente.Should().Be("KPG");
    }

    [Fact]
    public async Task Handle_WhenUserHasMultipleRegistros_ShouldReturnOrderedByFechaDesc()
    {
        await using var context = CreateContext();
        context.RegistrosHoras.Add(MakeRegistro("user-1", "A", "P1", new DateOnly(2026, 5, 10)));
        context.RegistrosHoras.Add(MakeRegistro("user-1", "B", "P2", new DateOnly(2026, 5, 12)));
        context.RegistrosHoras.Add(MakeRegistro("user-1", "C", "P3", new DateOnly(2026, 5, 11)));
        await context.SaveChangesAsync(CancellationToken.None);

        var handler = new GetMisRegistrosQueryHandler(context, new TestUser("user-1"));
        var result = (await handler.Handle(new GetMisRegistrosQuery(), CancellationToken.None)).ToList();

        result.Should().HaveCount(3);
        result[0].FechaRegistro.Should().Be(new DateOnly(2026, 5, 12));
        result[1].FechaRegistro.Should().Be(new DateOnly(2026, 5, 11));
        result[2].FechaRegistro.Should().Be(new DateOnly(2026, 5, 10));

    public async Task Handle_WhenRegistroExists_ShouldMapAllRequiredFields()
```

```csharp
public async Task<List<HistorialRegistroResponse>> GetHistorialAsync(CancellationToken cancellationToken = default)
{
    if (string.IsNullOrWhiteSpace(_authState.AccessToken))
        return [];

    using var httpRequest = new HttpRequestMessage(HttpMethod.Get, "api/registros-horas");
    httpRequest.Headers.Authorization = new AuthenticationHeaderValue("Bearer", _authState.AccessToken);

    var response = await _http.SendAsync(httpRequest, cancellationToken);
    if (!response.IsSuccessStatusCode)
        return [];

    return await response.Content.ReadFromJsonAsync<List<HistorialRegistroResponse>>(cancellationToken: cancellationToken)
           ?? [];
}
```

### MudDataGrid — `InitialDirection` para DateOnly

En MudBlazor 8.x, `PropertyColumn` con `InitialDirection="SortDirection.Descending"` establece el sort inicial. Solo una columna puede tener dirección inicial. Colocarlo en la columna `FechaRegistro` para que los registros más recientes aparezcan primero, alineado con el orden del backend (que ya ordena por fecha DESC).
