# Story 3.1: Aplicar Ventana de Registro Retroactivo

Status: review

## Story

As a empleado,
I want saber que fechas puedo registrar directamente,
so that pueda registrar dentro de la ventana permitida sin pedir ayuda.

## Acceptance Criteria

1. Given la ventana de retroactividad configurada en parametros del sistema, when el usuario abre el selector de fecha, then el sistema marca visualmente fechas disponibles, retroactivas permitidas y fuera de ventana, and la regla usa dias habiles segun configuracion, no un valor hardcodeado.
2. Given una fecha dentro de la ventana permitida, when el empleado registra horas para esa fecha, then el sistema permite guardar si el resto del formulario es valido, and registra que fue un registro retroactivo cuando aplique.
3. Given una fecha fuera de ventana, when el empleado intenta registrar directamente, then el sistema bloquea el guardado directo, and ofrece iniciar solicitud de excepcion (que se implementa en Story 3.2).

## Dev Notes

### Problema clave: IClock para tests determinísticos

Los tests existentes usan `new DateOnly(2026, 5, 14)` como fecha de referencia. Si el handler valida la ventana contra `DateTime.Today` directamente, los tests fallaran cuando se ejecuten en una fecha posterior. La solucion es inyectar `IClock` para que los tests puedan fijar la fecha de referencia con `TestClock`.
