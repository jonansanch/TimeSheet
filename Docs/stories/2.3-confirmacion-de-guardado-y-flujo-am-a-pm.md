# Story 2.3: Confirmación de Guardado y Flujo AM a PM

Status: done

## Story

As a empleado,
I want recibir confirmación clara después de guardar cada turno,
so that tenga certeza de que mis horas quedaron registradas correctamente.

## Acceptance Criteria

1. Dado un turno guardado exitosamente, cuando el sistema confirma la operación, entonces muestra un banner inline con turno, fecha, horas, cliente y proyecto, y el banner usa `role="status"` y `aria-live="polite"`.
2. Dado el turno AM guardado, cuando la confirmación se muestra, entonces el turno PM se activa automáticamente y aparece la opción de copiar configuración del AM.
3. Dados ambos turnos guardados, cuando el empleado está en la pantalla de registro, entonces la pantalla muestra "Jornada completa" con el total de horas registradas.

```razor
@implements IDisposable

@if (IsVisible)
{
    <MudPaper Elevation="2" Class="pa-3 mb-2" role="status" aria-live="polite">
        <MudStack Row="true" AlignItems="AlignItems.Center" Justify="Justify.SpaceBetween">
            <MudStack Row="true" AlignItems="AlignItems.Center" Spacing="2">
                <MudIcon Icon="@Icons.Material.Filled.CheckCircle" Color="Color.Success" />
                <div>
                    <MudText Typo="Typo.body1">
                        Turno @Turno guardado — @Fecha.ToString("dd/MM/yyyy")
                    </MudText>
                    <MudText Typo="Typo.body2" Color="Color.Secondary">
                        @HoraEntrada.ToString(@"HH\:mm") – @HoraSalida.ToString(@"HH\:mm")
                        | @Cliente | @Proyecto
                    </MudText>
                </div>
            </MudStack>
            <MudIconButton Icon="@Icons.Material.Filled.Close"
                           Size="Size.Small"
                           aria-label="Cerrar confirmación"
                           OnClick="Dismiss" />
        </MudStack>
    </MudPaper>
}
```

El `role="status"` y `aria-live="polite"` van en el `MudPaper` (elemento raíz del renderizado condicional). Usar `aria-live="polite"` para que los lectores de pantalla anuncien el mensaje al terminar la acción actual, no interrumpiendo.

### Auto-Dismiss con CancellationTokenSource

```csharp
private CancellationTokenSource? _cts;

protected override void OnParametersSet()
{
    if (IsVisible)
    {
        _cts?.Cancel();
        _cts = new CancellationTokenSource();
        _ = AutoDismissAsync(_cts.Token);
    }
}

private async Task AutoDismissAsync(CancellationToken token)
{
    try
    {
        await Task.Delay(8000, token);
        await InvokeAsync(async () =>
        {
            await OnDismiss.InvokeAsync();
        });
    }
    catch (OperationCanceledException) { }
}

private Task Dismiss() => OnDismiss.InvokeAsync();

public void Dispose() => _cts?.Cancel();
```

**Advertencia:** `_ = AutoDismissAsync(...)` inicia una tarea en background. Si el componente se desmonta antes de los 8 segundos, el `Dispose` cancela el CTS y la tarea termina limpiamente.

### Cálculo de Total de Horas

`TimeOnly.ToTimeSpan()` devuelve la hora como duración desde medianoche. La diferencia entre dos `TimeOnly` via `ToTimeSpan()` da la duración del turno:

```csharp
private string TotalHorasFormateado
{
    get
    {
        if (_amResponse is null || _pmResponse is null)
            return string.Empty;
        var durAM = _amResponse.HoraSalida.ToTimeSpan() - _amResponse.HoraEntrada.ToTimeSpan();
        var durPM = _pmResponse.HoraSalida.ToTimeSpan() - _pmResponse.HoraEntrada.ToTimeSpan();
        var total = durAM + durPM;
        return $"{(int)total.TotalHours:D2}:{total.Minutes:D2}";
    }
}
```

Esto es aceptable para esta story; Story 2.5 puede refinarlo.
