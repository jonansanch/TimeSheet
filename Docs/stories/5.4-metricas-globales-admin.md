# Story 5.4: Métricas Globales para Admin

Status: review

## Story

As an admin,
I want visualizar métricas globales del sistema,
So that pueda monitorear uso, tendencias y distribución general.

## Acceptance Criteria

1. **Given** un Admin autenticado, **when** abre métricas globales, **then** ve totales, distribuciones y tendencias del sistema **and** puede filtrar por período.

2. **Given** datos globales disponibles, **when** se cargan las métricas, **then** las consultas usan lecturas optimizadas con Dapper **and** la carga cumple el objetivo ≤ 3 segundos.

3. **Given** errores al cargar métricas, **when** ocurre un fallo interno, **then** se muestra mensaje de error no sensible **and** se registra contexto técnico en logs.

## Tasks / Subtasks

### Backend

- [ ] Crear `Application/Features/Dashboard/Queries/GetMetricasGlobales/` — totales registros, horas, usuarios activos, clientes activos con Dapper
- [ ] Agregar endpoint `GET /api/dashboard/admin?desde={}&hasta={}` (solo Admin)
