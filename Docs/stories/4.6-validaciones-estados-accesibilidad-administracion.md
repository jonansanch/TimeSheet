# Story 4.6: Validaciones, Estados y Accesibilidad de Administración

Status: review

## Story

As an admin,
I want pantallas de administración claras, accesibles y consistentes,
So that pueda mantener datos operativos sin errores y sin necesitar el mouse.

## Acceptance Criteria

1. **Given** cualquier tabla administrativa, **when** carga datos, **then** muestra skeleton rows durante la carga (no solo barra lineal) **and** muestra estados vacíos específicos al catálogo cuando no hay registros.

2. **Given** formularios de administración (diálogos), **when** el Admin los abre, **then** el foco se posiciona automáticamente en el primer campo interactivo **and** puede completar, guardar (Enter/Tab+Space) y cancelar (Escape) sin mouse; el foco visible se conserva en todo momento.

3. **Given** acciones destructivas o desactivaciones en cualquier página admin, **when** el Admin las ejecuta, **then** el sistema muestra confirmación específica con `MudMessageBox` y botón con color indicativo (Warning/Error) **and** los datos históricos se conservan mediante soft delete.

4. **Given** la página de Usuarios (`/admin/usuarios`), **when** hay más de 10 usuarios, **then** el Admin puede filtrar por email usando un campo de búsqueda (QuickFilter) similar al de Empleados y Clientes.

5. **Given** alertas de error en páginas y diálogos admin, **when** aparecen, **then** tienen `aria-live="polite"` para que lectores de pantalla las anuncien sin interrumpir al usuario.

## Dev Notes

### Componente KpgTableSkeleton — patrón exacto a crear

```razor
@* Fronted/src/WebUI/Shared/Components/KpgTableSkeleton.razor *@
<MudStack Spacing="1">
    @for (var i = 0; i < Rows; i++)
    {
        <MudSkeleton Width="100%" Height="48px" SkeletonType="SkeletonType.Rectangle" />
    }
</MudStack>

@code {
    [Parameter] public int Rows { get; set; } = 5;
}
```

`KpgTableSkeleton` ya está registrado globalmente por `_Imports.razor` (`@using KPG.Timesheet.WebUI.Shared.Components`). No necesita `@using` adicional en las páginas.

### Reemplazo de MudProgressLinear — patrón

Antes (actual en todas las páginas admin):
```razor
@if (_isLoading)
{
    <MudProgressLinear Indeterminate="true" Size="Size.Small" />
}
```

Después:
```razor
@if (_isLoading)
{
    <KpgTableSkeleton />
}
```

### aria-live en MudAlert de error — patrón

Antes:
```razor
<MudAlert Severity="Severity.Error" Dense="true">@_errorMessage</MudAlert>
```

Después:
```razor
<MudAlert Severity="Severity.Error" Dense="true" aria-live="polite">@_errorMessage</MudAlert>
```

Nota: `role="status"` ya está en los MudAlert de success — no cambiar eso.

### FocusAsync en diálogos — patrón exacto (tomar de KpgUserDialog que ya lo implementa)

```razor
@* En el template *@
<MudTextField @ref="_nombreField"
              T="string"
              Label="Nombre"
              .../>

@* En @code *@
private MudTextField<string> _nombreField = default!;

protected override async Task OnAfterRenderAsync(bool firstRender)
{
    if (firstRender)
        await _nombreField.FocusAsync();
}
```

`KpgUserDialog` ya usa este mismo patrón en `_emailField` — la implementación ya es correcta.

### QuickFilter en UsuariosAdminPage — patrón (copiar de EmpleadosAdminPage)

Agregar campo de búsqueda en @code:
```csharp
private string _searchText = string.Empty;
private Func<UserAdminResponse, bool> _quickFilter =>
    u => string.IsNullOrWhiteSpace(_searchText)
        || u.Email.Contains(_searchText, StringComparison.OrdinalIgnoreCase);
```

Agregar al MudDataGrid:
```razor
<MudDataGrid T="UserAdminResponse"
             Items="_usuarios"
             QuickFilter="_quickFilter"
             ...>

        <MudSpacer />

                      AdornmentIcon="@Icons.Material.Filled.Search"
                      IconSize="Size.Small"
                      Immediate="true"
                      Style="max-width:280px;" />

    ...
</MudDataGrid>
```
