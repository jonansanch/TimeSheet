# Story 5.1: Dashboard de Estado Diario del Equipo

Status: review

## Story

As a supervisor,
I want ver quién registró y quién está pendiente hoy,
So that pueda actuar antes del cierre del día.

## Acceptance Criteria

1. **Given** un Supervisor autenticado, **when** abre el dashboard, **then** ve el número de pendientes como métrica principal sin scroll **and** ve el estado diario de cada miembro del equipo.

2. **Given** miembros con registros completos, parciales o pendientes, **when** se renderiza la lista, **then** cada colaborador aparece con nombre, iniciales (avatar), horas del día y badge de estado **and** los estados distinguen Completo, Parcial y Pendiente.

3. **Given** el dashboard cargando, **when** se solicitan datos, **then** muestra skeleton **and** la carga objetivo es ≤ 3 segundos.

4. **Given** un Gerente o Admin autenticado, **when** abre el dashboard, **then** también puede ver el estado diario del equipo completo.

5. **Given** el selector de fecha, **when** el Supervisor cambia de fecha, **then** la lista se recarga con los registros de esa fecha.

  ```csharp
  [Authorize(Roles = $"{Roles.Supervisor},{Roles.Gerente},{Roles.Admin}")]

  ```
- [ ] Crear `GetEstadoEquipoQueryHandler.cs` con Dapper usando SQL:
  ```sql
  SELECT u.Id AS UserId,
         ISNULL(u.NombreCompleto, u.Email) AS Nombre,
         u.Email,
         MAX(CASE WHEN r.Turno = 1 THEN 1 ELSE 0 END) AS TieneAM,
         MAX(CASE WHEN r.Turno = 2 THEN 1 ELSE 0 END) AS TienePM
  FROM   AspNetUsers u
  JOIN   AspNetUserRoles ur ON u.Id = ur.UserId
  JOIN   AspNetRoles ro     ON ur.RoleId = ro.Id
  LEFT   JOIN RegistrosHoras r ON r.UserId = u.Id AND r.FechaRegistro = @Fecha
  WHERE  u.IsActive = 1
    AND  ro.Name IN ('Empleado', 'Supervisor')
  GROUP  BY u.Id, u.NombreCompleto, u.Email
  ORDER  BY ISNULL(u.NombreCompleto, u.Email)
  ```
  - Estado = "Completo" si TieneAM=1 && TienePM=1; "Parcial" si exactamente uno; "Pendiente" si ninguno.

## Dev Notes

### Decisión de diseño: equipo = todos los empleados activos (V1)

No existe mapping supervisor→empleados en V1. El dashboard muestra TODOS los usuarios activos con rol `Empleado` o `Supervisor`. Esto es correcto para 30 usuarios con una organización plana.

### NombreCompleto en ApplicationUser

`ApplicationUser` solo tiene `Email` como identificador legible. La UI del dashboard muestra nombres ("Ana García"), no emails. Se agrega `NombreCompleto` nullable a `ApplicationUser`:
- Si es null, el handler y la query usan `Email` como fallback (`ISNULL(NombreCompleto, Email)`)
- Campo opcional en CreateUser — el Admin puede dejar en blanco (fallback = email)
- No se actualiza en batches los usuarios existentes; quedarán con NombreCompleto = null (mostrarán email en el dashboard hasta que se editen)

### Dapper: patrón de inyección

Inyectar `IDbConnection` en el handler (registrado como scoped):
```csharp
public class GetEstadoEquipoQueryHandler(IDbConnection db)
    : IRequestHandler<GetEstadoEquipoQuery, EstadoEquipoResponse>
```
NO usar `SqlConnection` directamente en el handler — respetar la abstracción de `IDbConnection`.

### TurnoRegistro: AM = 1, PM = 2

El SQL filtra con `r.Turno = 1` para AM y `r.Turno = 2` para PM.

### KpgStatCard — variante de color

No usar el enum de variantes (Default/Alert/Success como string) — usar `MudBlazor.Color` directamente:
- Completos → `Color.Success`
- Parciales → `Color.Warning`
- Pendientes → `Color.Error`
- Total → `Color.Primary`

### KpgTeamStatusCard — avatar de iniciales

Para generar las iniciales: `string.Join("", nombre.Split(' ').Take(2).Select(p => p[0]))`.

### Selector de fecha en dashboard

Usar `MudDatePicker` con `@bind-Date="_fecha"` donde `_fecha` es `DateTime?`. Al cambiar fecha, llamar `await CargarAsync()`.
