# Story 5.5: Reporte de Horas con Filtros

Status: review

## Story

As a supervisor,
I want generar reportes filtrados por período, empleado, cliente o proyecto,
So that pueda preparar el cierre mensual sin consolidación manual.

## Acceptance Criteria

1. **Given** un Supervisor autenticado, **when** abre reportes, **then** puede filtrar por día, semana, mes o rango personalizado **and** puede filtrar por empleado, cliente o proyecto dentro de su alcance.

2. **Given** filtros válidos, **when** genera el reporte, **then** el sistema devuelve resultados en menos de 10 segundos **and** muestra registros con fecha, empleado, cliente, proyecto, horas y descripción.

3. **Given** filtros sin resultados, **when** genera el reporte, **then** se muestra estado vacío específico **and** permite ajustar filtros sin perder la selección actual.

- [ ] Crear handler con Dapper — query dinámica construida con parámetros opcionales
  ```sql
  SELECT u.Email, ISNULL(u.NombreCompleto, u.Email) AS NombreEmpleado,
         r.FechaRegistro, r.Turno, r.HoraEntrada, r.HoraSalida,
         r.Cliente, r.Proyecto, r.Modalidad, r.Descripcion, r.Lugar,
         DATEDIFF(MINUTE, r.HoraEntrada, r.HoraSalida) / 60.0 AS Horas
  FROM   RegistrosHoras r
  JOIN   AspNetUsers u ON r.UserId = u.Id
  WHERE  r.FechaRegistro BETWEEN @Desde AND @Hasta
    AND  (@UserId IS NULL OR r.UserId = @UserId)
    AND  (@Cliente IS NULL OR r.Cliente = @Cliente)
    AND  (@Proyecto IS NULL OR r.Proyecto = @Proyecto)
  ORDER  BY r.FechaRegistro DESC, NombreEmpleado
  ```

## Dev Notes

### Query dinámica con Dapper

Para los filtros opcionales, pasar siempre los parámetros (usando `null` cuando no aplica) y escribir el WHERE con `AND (@Param IS NULL OR Campo = @Param)`. Esto permite usar query preparada sin SQL dinámico.

### Paginación del reporte

El reporte puede tener muchas filas. Usar `MudDataGrid` con `RowsPerPage="25"` y selector 25/50/100. No paginar en servidor en V1 (cargar hasta 1000 filas, suficiente para 30 usuarios × 30 días).

### Filtro de período por defecto

Al cargar: mes actual (primer día del mes → hoy).
