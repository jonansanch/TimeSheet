# Story 5.10: Enviar Correos Automáticos por Registros Pendientes

Status: done

## Story

As a empleado or consultor,
I want recibir un correo cuando llevo varios días sin registrar horas,
So that pueda corregir el atraso antes del cierre mensual.

## Acceptance Criteria

1. **Given** un empleado supera el umbral de días sin registro, **when** corre el proceso de notificación, **then** el sistema envía un correo al empleado **and** el correo indica días pendientes y acción esperada.

2. **Given** configuración SMTP válida, **when** el proceso envía correos, **then** registra resultado de envío por destinatario **and** no duplica notificaciones para el mismo pendiente dentro de la ventana configurada.

3. **Given** falla el envío de un correo, **when** el proveedor retorna error, **then** el sistema registra el fallo con contexto técnico **and** no interrumpe el procesamiento de otros destinatarios.

## Dev Notes

### SMTP en desarrollo

Para desarrollo, usar MailHog o Mailtrap (SMTP fake). Configurar en `appsettings.Development.json`.

### Template de correo

Texto plano en V1 (sin HTML fancy):
```
Estimado/a [Nombre],

Notamos que llevas [N] días sin registrar tus horas en KPG Timesheet.

Por favor ingresa al sistema y registra tus horas pendientes para evitar inconvenientes en el cierre mensual.

KPG Timesheet — Sistema de Gestión de Horas
```

### Evitar duplicados

Verificar: `SELECT COUNT(*) FROM NotificacionesEnviadas WHERE UserId = @UserId AND Created >= DATEADD(HOUR, -24, GETUTCDATE())`. Si > 0, omitir envío.
