# Story 5.10: Enviar Correos Automáticos por Registros Pendientes

Status: draft

## Story

As a empleado or consultor,
I want recibir un correo cuando llevo varios días sin registrar horas,
So that pueda corregir el atraso antes del cierre mensual.

## Acceptance Criteria

1. **Given** un empleado supera el umbral de días sin registro, **when** corre el proceso de notificación, **then** el sistema envía un correo al empleado **and** el correo indica días pendientes y acción esperada.

2. **Given** configuración SMTP válida, **when** el proceso envía correos, **then** registra resultado de envío por destinatario **and** no duplica notificaciones para el mismo pendiente dentro de la ventana configurada.

3. **Given** falla el envío de un correo, **when** el proveedor retorna error, **then** el sistema registra el fallo con contexto técnico **and** no interrumpe el procesamiento de otros destinatarios.

## Tasks / Subtasks

### Backend — Tabla NotificacionesEnviadas

- [ ] Crear entidad `NotificacionEnviada` en Domain:
  ```csharp
  public class NotificacionEnviada : BaseAuditableEntity
  {
      public string UserId { get; set; }
      public string Email { get; set; }
      public DateOnly FechaReferencia { get; set; }  // último día sin registrar
      public int DiasAcumulados { get; set; }
      public bool Exitoso { get; set; }
      public string? ErrorDetalle { get; set; }
  }
  ```
- [ ] Agregar `DbSet<NotificacionEnviada>` al DbContext
- [ ] Crear migration

### Backend — Email Service

- [ ] Crear `IEmailService` interface en Application.Common.Interfaces:
  ```csharp
  Task<bool> SendAsync(string to, string subject, string body);
  ```
- [ ] Crear `SmtpEmailService` en Infrastructure usando `MailKit` o `System.Net.Mail.SmtpClient`
- [ ] Configurar SMTP en `appsettings.json` (SmtpHost, SmtpPort, FromAddress, FromName)
- [ ] Registrar en DI

### Backend — BackgroundService

- [ ] Crear `NotificacionesPendientesJob` como `BackgroundService` o `IHostedService`
  - Corre diariamente a las 8:00 AM
  - Lee umbral de `ParametrosSistema`
  - Obtiene usuarios con N+ días sin registro (reutiliza lógica de 5.9)
  - Para cada usuario: verifica que no se le envió notificación en las últimas 24h (evitar duplicados)
  - Envía correo con `IEmailService`
  - Registra resultado en `NotificacionesEnviadas`
- [ ] Registrar en `Program.cs`: `builder.Services.AddHostedService<NotificacionesPendientesJob>()`

### Verificación

- [ ] Build limpio
- [ ] Test manual: ajustar umbral a 0 días y verificar que el job genera notificaciones (o simular ejecución manual via endpoint de debug solo en dev)

## Dev Notes

### SMTP en desarrollo

Para desarrollo, usar MailHog o Mailtrap (SMTP fake). Configurar en `appsettings.Development.json`.

### Template de correo

Texto plano en V1 (sin HTML fancy):
```
Estimado/a [Nombre],

Notamos que llevas [N] días sin registrar tus horas en KPG Timesheet.

Por favor ingresa al sistema y registra tus horas pendientes para evitar inconvenientes en el cierre mensual.

KPG Timesheet — Sistema de Gestión de Horas
```

### Evitar duplicados

Verificar: `SELECT COUNT(*) FROM NotificacionesEnviadas WHERE UserId = @UserId AND Created >= DATEADD(HOUR, -24, GETUTCDATE())`. Si > 0, omitir envío.
