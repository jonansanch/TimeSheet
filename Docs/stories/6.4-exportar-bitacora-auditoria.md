# Story 6.4: Exportar Bitácora para Auditoría

Status: review

## Story

As an admin,
I want exportar bitácoras filtradas a Excel,
So that pueda entregar evidencia digital a clientes o dirección.

## Acceptance Criteria

1. **Given** un Admin con la bitácora cargada (resultado de la búsqueda), **when** pulsa "Exportar Excel", **then** el sistema genera y descarga un archivo `.xlsx` con los mismos eventos visibles en la tabla (respetando los filtros activos).

2. **Given** la exportación generada, **when** se abre el archivo, **then** contiene las columnas: Fecha y hora, Tipo de evento, Actor, Entidad afectada, ID entidad, Metadata. El campo Metadata incluye el JSON técnico completo (sin filtrar — es para auditoría), pero no expone tokens, passwords ni secretos de sesión (la columna ya no los contiene por diseño del modelo).

3. **Given** la exportación generada, **when** se revisa el nombre del archivo, **then** sigue el patrón `bitacora-YYYYMMDD-YYYYMMDD.xlsx` (fechas del filtro activo, o fecha de hoy si no hay filtros).

4. **Given** la bitácora con 0 resultados o sin búsqueda previa, **when** el Admin ve la página, **then** el botón "Exportar Excel" está deshabilitado.

5. **Given** una exportación en curso, **when** el usuario espera, **then** el botón muestra un spinner (`MudProgressCircular`) y queda deshabilitado hasta completar.

6. **Given** un error al generar la exportación, **when** ocurre, **then** la UI muestra un `MudAlert` con Severity.Error con mensaje no sensible y sin detalles técnicos internos.

7. **Given** un usuario sin rol Admin, **when** llama a `GET /api/bitacora/excel`, **then** recibe `403 Forbidden`.

    ```sql
    SELECT b.Id,
           b.TipoEvento,
           b.ActorId,
           b.ActorEmail,
           ISNULL(u.NombreCompleto, b.ActorEmail) AS ActorNombre,
           b.EntidadAfectada,
    ```

  Estructura del handler:
  ```csharp
  using System.Data;
  using Dapper;
  using KPG.Timesheet.Application.Features.Bitacora.Queries.ExportarBitacora;
  using MediatR;
  using MiniExcelLibs;

  namespace KPG.Timesheet.Infrastructure.Bitacora;

  public class ExportarBitacoraQueryHandler(IDbConnection db)
      : IRequestHandler<ExportarBitacoraQuery, ExportarBitacoraResult>
  {
      private const string Sql = """..."""; // mismo SQL del GetBitacoraQueryHandler

      public async Task<ExportarBitacoraResult> Handle(
          ExportarBitacoraQuery request,
          CancellationToken cancellationToken)
      {
          var rows = (await db.QueryAsync<RawRow>(Sql, new
          {
              Desde      = request.Desde,
              Hasta      = request.Hasta,
              ActorId    = request.ActorId,
              TipoEvento = request.TipoEvento
          })).ToList();

          var exportRows = rows.Select(r => new
          {
              FechaHora         = r.Timestamp.LocalDateTime.ToString("dd/MM/yyyy HH:mm"),
              TipoEvento        = r.TipoEvento,
              Actor             = r.ActorNombre ?? r.ActorEmail ?? r.ActorId,
              EntidadAfectada   = r.EntidadAfectada,
              EntidadId         = r.EntidadId ?? string.Empty,
              Metadata          = r.MetadataJson ?? string.Empty
          }).ToList();

          using var ms = new MemoryStream();
          ms.SaveAs(exportRows);

          var hoy = DateOnly.FromDateTime(DateTime.Today);
          var desde = request.Desde ?? hoy;
          var hasta  = request.Hasta  ?? hoy;
          var fileName = $"bitacora-{desde:yyyyMMdd}-{hasta:yyyyMMdd}.xlsx";

          return new ExportarBitacoraResult(
              ms.ToArray(),
              "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet",
              fileName);
      }

      // record interno para mapeo Dapper (reutiliza mismos campos que BitacoraItemDto)
      private sealed record RawRow(
          int Id,
          string TipoEvento,
          string ActorId,
          string? ActorEmail,
          string? ActorNombre,
          string EntidadAfectada,
          string? EntidadId,
          DateTimeOffset Timestamp,
          string? MetadataJson);
  }
  ```

    ```csharp
    public async Task<(byte[] Contenido, string ContentType, string FileName)?> ExportarExcelAsync(
        DateOnly? desde = null,
        DateOnly? hasta = null,
        string? actorId = null,
        string? tipoEvento = null,
        CancellationToken ct = default)
    {
        if (string.IsNullOrWhiteSpace(_authState.AccessToken))
            throw new UnauthorizedAccessException("No hay token de acceso activo.");

        var sb = new StringBuilder("api/bitacora/excel?");
        if (desde.HasValue)                         sb.Append($"desde={desde.Value:yyyy-MM-dd}&");
        if (hasta.HasValue)                         sb.Append($"hasta={hasta.Value:yyyy-MM-dd}&");
        if (!string.IsNullOrWhiteSpace(actorId))    sb.Append($"actorId={Uri.EscapeDataString(actorId)}&");
        if (!string.IsNullOrWhiteSpace(tipoEvento)) sb.Append($"tipoEvento={Uri.EscapeDataString(tipoEvento)}&");

        using var request = new HttpRequestMessage(HttpMethod.Get, sb.ToString().TrimEnd('&', '?'));
        request.Headers.Authorization = new AuthenticationHeaderValue("Bearer", _authState.AccessToken);

        var response = await _http.SendAsync(request, ct);
        if (!response.IsSuccessStatusCode) return null;

        var bytes      = await response.Content.ReadAsByteArrayAsync(ct);
        var contentType = response.Content.Headers.ContentType?.MediaType
                        ?? "application/vnd.openxmlformats-officedocument.spreadsheetml.sheet";
        var fileName   = response.Content.Headers.ContentDisposition?.FileNameStar
                      ?? response.Content.Headers.ContentDisposition?.FileName
                      ?? "bitacora.xlsx";

        return (bytes, contentType, fileName.Trim('"'));
    }
    ```

### MiniExcel — ya instalado, mismo patrón

MiniExcel ya está como dependencia del proyecto Backend (ver `ExportarReporteHorasQueryHandler`). El patrón es exactamente:
```csharp
using MiniExcelLibs;
using var ms = new MemoryStream();
ms.SaveAs(rows); // rows = IEnumerable de objetos anónimos con propiedades = columnas
var bytes = ms.ToArray();
```
`ms.SaveAs(rows)` infiere los nombres de columna de los nombres de propiedades del objeto. Los nombres de propiedad que se pasen se vuelven encabezados del Excel — usar nombres descriptivos en español (FechaHora, TipoEvento, Actor, EntidadAfectada, EntidadId, Metadata).

**No usar ClosedXML aquí** — MiniExcel genera tabular plano, que es suficiente para auditoría. ClosedXML es para formatos estilizados (timesheet). La bitácora no necesita formato complejo.

### `Results.File` + Content-Disposition en el endpoint

El endpoint usa `Results.File(result.Contenido, result.ContentType, result.FileName)`. Minimal APIs con `Results.File` automáticamente agrega el header `Content-Disposition: attachment; filename*=UTF-8''<fileName>`. El frontend lee `ContentDisposition?.FileNameStar` primero (UTF-8 encoding), luego `FileName` como fallback. Este patrón ya funciona en `ExportarTimesheetAsync`.
