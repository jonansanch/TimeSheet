# Story 2.2: Seleccionar Cliente, Proyecto y Contexto con Sugerencias Recientes

Status: done

## Story

As a empleado,
I want seleccionar rápidamente cliente, proyecto y contexto de trabajo,
so that pueda completar mi registro diario en menos de 2 minutos.

## Acceptance Criteria

1. Dado un empleado con registros previos, cuando abre el formulario de registro, entonces ve las últimas 3–5 combinaciones cliente/proyecto usadas y puede aplicar una sugerencia con clic o teclado.
2. Dado un cliente seleccionado, cuando el empleado abre el selector de proyecto, entonces solo se muestran proyectos asociados a ese cliente (derivados de sugerencias recientes o historial) y el selector permite búsqueda integrada.
3. Dado el campo modalidad, cuando el empleado navega por las opciones, entonces puede seleccionar modalidad mediante chips accesibles y los chips funcionan con teclado como grupo de opción única (radiogroup).

### Decisión de Diseño: Filtrado de Proyectos sin Catálogos

Los catálogos reales no existen hasta Epic 4. La estrategia para UX-DR18 (filtrar proyectos por cliente) es:
1. Al cargar el componente, obtener sugerencias recientes del API → lista de tuplas `(Cliente, Proyecto)`.
2. En el `SearchFunc` del autocomplete de Proyecto: si `model.Cliente` está completo, filtrar las sugerencias por ese cliente y devolver sus proyectos. Si no hay coincidencias o no hay cliente seleccionado, devolver todos los valores del array estático `Proyectos`.
3. Esto cumple UX-DR18 con datos reales del usuario sin necesitar catálogos.

### Endpoint de Sugerencias Recientes

```
GET /api/registros-horas/recientes?top=5
Authorization: Bearer {jwt}
```

Respuesta exitosa (200):
```json
[
  { "cliente": "KPG", "proyecto": "Timesheet" },
  { "cliente": "Cliente externo", "proyecto": "Consultoria" }
]
```

- `top` es query param opcional, default 5, máximo 10.
- El `userId` viene de `ICurrentUser.Id` (no de query param — seguridad).
- Si el usuario no tiene registros previos, retorna array vacío `[]` — no es un error.
- Query LINQ: `.Where(r => r.UserId == userId).GroupBy(r => new { r.Cliente, r.Proyecto }).OrderByDescending(g => g.Max(r => r.FechaRegistro)).Take(top).Select(g => new RegistroRecienteDto(g.Key.Cliente, g.Key.Proyecto))`.

### MudAutocomplete — Patrón Correcto MudBlazor

```razor
<MudAutocomplete T="string"
                 Label="Cliente"
                 Variant="Variant.Outlined"
                 @bind-Value="model.Cliente"
                 SearchFunc="@((text, token) => SearchClientesAsync(text, token))"
                 Required="true"
                 RequiredError="El cliente es requerido."
                 Immediate="false"
                 CoerceValue="true" />
```

- `CoerceValue="true"` permite que el usuario escriba un valor libre (no solo de la lista) — importante porque los catálogos son strings libres en esta etapa.
- `Immediate="false"` evita búsqueda en cada tecla.
- `SearchFunc` debe ser `Func<string, CancellationToken, Task<IEnumerable<string>>>`.

```csharp
private async Task<IEnumerable<string>> SearchClientesAsync(string text, CancellationToken token)
{
    var todos = _sugerenciasRecientes.Select(s => s.Cliente)
        .Concat(Clientes)
        .Distinct(StringComparer.OrdinalIgnoreCase);
    if (string.IsNullOrWhiteSpace(text)) return todos;
    return todos.Where(c => c.Contains(text, StringComparison.OrdinalIgnoreCase));
}
```

### MudChipSet — Patrón para Modalidad

```razor
<MudChipSet T="string"
            SelectionMode="SelectionMode.SingleSelection"
            @bind-SelectedValue="model.Modalidad"
            aria-label="Modalidad de trabajo">
    @foreach (var modalidad in Modalidades)
    {
        <MudChip T="string" Value="@modalidad">@modalidad</MudChip>
    }
</MudChipSet>
```

- Validación: si `model.Modalidad` es null, el chip no tiene selección. El botón Guardar ya verifica `CanSave`, que requiere que todos los campos del modelo estén completos — agregar `!string.IsNullOrWhiteSpace(model.Modalidad)` a `CanSave` si no está ya incluido.
- Nota: `MudChipSet` con `SelectionMode.SingleSelection` y `@bind-SelectedValue` es el patrón de MudBlazor 7.x. Verificar versión en `Directory.Packages.props` antes de implementar.

### KpgRecentSuggestions — Patrón de Componente

```razor
@inject IRegistroHorasRepository RegistroHorasRepository

@if (_sugerencias.Any())
{
    <MudStack Row="true" Wrap="Wrap.Wrap" Spacing="1">
        <MudText Typo="Typo.caption" Color="Color.Secondary" Class="align-self-center">Usados recientemente:</MudText>
        @foreach (var s in _sugerencias)
        {
            <MudChip T="string"
                     OnClick="@(() => OnSuggestionSelected.InvokeAsync((s.Cliente, s.Proyecto)))"
                     aria-label="@($"Aplicar: {s.Cliente} - {s.Proyecto}")"
                     Color="Color.Default"
                     Variant="Variant.Outlined"
                     Size="Size.Small">
                @s.Cliente — @s.Proyecto
            </MudChip>
        }
    </MudStack>
}

@code {
    [Parameter] public EventCallback<(string Cliente, string Proyecto)> OnSuggestionSelected { get; set; }
    private List<RegistroRecienteResponse> _sugerencias = [];

    protected override async Task OnInitializedAsync()
    {
        try { _sugerencias = await RegistroHorasRepository.GetRecientesAsync(5) ?? []; }
        catch { /* silencioso — no bloquear formulario */ }
    }
}
```

private void HandleSuggestionSelected((string Cliente, string Proyecto) sugerencia)
